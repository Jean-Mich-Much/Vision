🚀 Vision – Natural Data Engine (NDE)

Vision is a Natural Data Engine (NDE). An NDE is an evolution of traditional databases: it stores, reads, and links data like a classic database, but without tables, without schema, without SQL, and without any imposed structure.

Vision is built on two foundations:
• UniCell — every piece of data is an autonomous cell stored in a .vr file
• Wave — a minimal global state that coordinates the engine

Unlike traditional databases, Vision does not predict, cascade, or restructure anything automatically. It gives full control to the developer while remaining stable and fully predictable.

1. Introduction

Vision is a lightweight data engine for PHP 8.3+, UTF‑8 LF, with zero dependencies. It is autonomous, simple, stable, and predictable.

Vision naturally reconstructs complex structures such as forums, CMS, knowledge graphs, presentations, scientific datasets, and more.

UniCell

Each piece of data is stored in a standalone .vr file.

Relation levels:

  • 0 → lateral
  • 1 → direct internal
  • 2 → deep internal
  • 3+ → rare cases

Wave

Wave is a minimal global state updated continuously. Modules read Wave and react locally. No prediction. No automatic cascade.

2. File Architecture

/vision/
    vision.php              → entry point (loads all modules)
    Vision_UniCell.php      → UniCell storage (.vr) + sharding
    Vision_API.php          → full Vision API

    vision_wave.php         → Wave global state
    vision_lock.php         → non-blocking global lock
    vision_worker.php       → async multitasking (Fibers)
    vision_autoclean.php    → autonomous maintenance

    /data/
        YYYYMMDD/
            id_unique.vr    → 1 file = 1 cell

3. UniCell – .vr Format

Each cell is stored in a .vr file:

VISION1
⟪¦<base64(json)>¦⟫
⟪⇒¦⇐⟫

Internal JSON:

{
  "id": "20260416_ab12cd34ef56",
  "data": {...},
  "relations": [
      {"id": "xxx", "level": 1},
      {"id": "yyy", "level": 0}
  ],
  "createdAt": 1713200000,
  "updatedAt": 1713200000
}

Properties:

  • strict atomicity: .tmp → flush → rename()
  • no partial writes
  • final marker validation
  • deterministic format
  • UTF‑8 LF

4. Sharding

Each cell is stored in:

/vision/data/YYYYMMDD/id.vr

The ID prefix (YYYYMMDD) automatically determines the shard.

5. Wave – Global State

Wave is a simple associative array:

[
  'charge'   => float,
  'priority' => string,
  'pressure' => string,
  'lock'     => string,
  'ram'      => string,
  'disk'     => string,
  'op'       => string
]

Each module reads Wave and reacts locally.

6. Internal Modules

9. Security & Robustness

  • strict atomicity
  • no partial writes
  • crash‑safe
  • non‑blocking global lock
  • no prediction
  • silent logging

10. Explicit Limitations

Vision will never implement:

  • transactions
  • triggers
  • prediction
  • automatic reorganization
  • schema inference
  • automatic indexes

11. Why Vision?

Vision is not SQL, not a document store, and not a classic graph database.

Compared to SQL

  • no tables, no schema
  • no joins
  • explicit relations

Compared to Document Stores

  • each cell is autonomous
  • native relations
  • date‑based sharding

Compared to Graph Databases

  • simple relation levels
  • cycle‑resistant

12. Conclusion

Vision is a natural, simple, stable, and predictable data engine. UniCell provides universality. Wave provides coherence. Modules provide dynamics.
