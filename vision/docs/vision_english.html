🚀 Vision - Natural Data Engine (NDE)

Vision is a Natural Data Engine (NDE).
An NDE is an evolution of traditional databases: it stores, reads and links data like a classic database, but it does so without tables, without schema, without SQL, and without imposed structure.

Vision is built on two foundations: UniCell, where each piece of data is an autonomous cell stored in a .vr file, and Wave, a minimal global state that coordinates the engine.

Unlike traditional databases, Vision does not predict, cascade or restructure anything automatically. It gives full control to the developer while remaining stable and fully predictable.

1. Introduction

Vision is a lightweight data engine for PHP 8.3+, UTF‑8 LF, with zero dependencies. It is autonomous, simple, stable and predictable. Vision naturally reconstructs complex structures such as forums, CMS, knowledge graphs, presentations, scientific datasets, and more.

  • UniCell - each piece of data is a standalone cell stored in a .vr file. Relation levels:
    • 0 = lateral
    • 1 = direct internal
    • 2 = deep internal
    • 3+ = rare cases
  • Wave - a minimal global state updated continuously. Modules read Wave and react locally. No prediction. No automatic cascade.

2. File Architecture

/vision/
    vision.php              → entry point (loads all modules)
    Vision_UniCell.php      → UniCell storage (.vr) + sharding
    Vision_API.php          → full Vision API

    vision_wave.php         → Wave global state
    vision_lock.php         → non-blocking global lock
    vision_worker.php       → async multitasking (Fibers)
    vision_autoclean.php    → autonomous maintenance

    /data/
        YYYYMMDD/
            id_unique.vr    → 1 file = 1 cell

3. UniCell - .vr Format

Each cell is stored in a .vr file:

VISION1
⟪¦<base64(json)>¦⟫
⟪⇒¦⇐⟫

Internal JSON:

{
  "id": "20260416_ab12cd34ef56",
  "data": {...},
  "relations": [
      {"id": "xxx", "niveau": 1},
      {"id": "yyy", "niveau": 0}
  ],
  "createdAt": 1713200000,
  "updatedAt": 1713200000
}

Properties:

  • strict atomicity: .tmp → flush → rename()
  • no partial writes
  • final marker validation
  • deterministic format
  • UTF‑8 LF

4. Sharding

Each cell is stored in:

/vision/data/YYYYMMDD/id.vr

The ID prefix (YYYYMMDD) automatically determines the shard.

5. Wave - Global State

Wave is a simple associative array:

[
  'charge'   => float,   // 0.0 → 1.0
  'priority' => string,  // read | write | clean | idle
  'pressure' => string,  // low | medium | high
  'lock'     => string,  // free | busy
  'ram'      => string,  // stable | high | critical
  'disk'     => string,  // idle | busy
  'op'       => string   // operation type
]

Each module reads Wave and reacts locally.

6. Internal Modules

vision.php

  • entry point
  • loads all modules
  • silent logging (vision.log)
  • no browser output

Vision_UniCell.php

  • handles .vr files
  • atomic read/write
  • format validation
  • date-based sharding

Vision_API.php

  • full public API
  • CRUD
  • relations
  • graphs
  • search
  • stats
  • maintenance

vision_wave.php

  • global state
  • load, pressure, RAM, disk
  • operation priority

vision_lock.php

  • non-blocking global lock
  • minimal internal queue
  • strict atomicity
  • no deadlocks

vision_worker.php

  • async multitasking (Fibers)
  • no busy waiting
  • no blocking loops

vision_autoclean.php

  • autonomous maintenance
  • runs only when: disk=idle, lock=free, charge<0.5
  • cleans orphan/corrupted files

7. Vision API (Complete)

8. Usage Examples

Forum

$topic = Vision_API::create(['title' => 'Hello']);

$post1 = Vision_API::create(
    ['text' => 'Hi'],
    [['id' => $topic, 'level' => 1]]
);

$post2 = Vision_API::create(
    ['text' => 'Welcome'],
    [['id' => $topic, 'level' => 1]]
);

$reply = Vision_API::create(
    ['text' => 'Thanks'],
    [['id' => $post1, 'level' => 1]]
);

CMS

$page = Vision_API::create(['title' => 'Home']);

$section = Vision_API::create(
    ['title' => 'Intro'],
    [['id' => $page, 'level' => 1]]
);

$block = Vision_API::create(
    ['text' => 'Welcome'],
    [['id' => $section, 'level' => 1]]
);

Knowledge Graph

$paris  = Vision_API::create(['name' => 'Paris']);
$france = Vision_API::create(['name' => 'France']);

Vision_API::link($paris, $france, 1);

Custom Search

$results = Vision_API::find(function(array $cell) {
    return isset($cell['data']['title'])
        && str_contains($cell['data']['title'], 'Hello');
});

9. Security & Robustness

  • strict atomicity (.tmp then rename())
  • no partial writes
  • no corruption possible
  • crash‑safe
  • non‑blocking global lock
  • no automatic cascade
  • no prediction
  • no browser output
  • silent logging

10. Explicit Limitations

Vision will never implement:

  • multi‑cell transactions
  • triggers
  • prediction
  • automatic global reorganization
  • content analysis
  • schema inference
  • forced hierarchy
  • automatic secondary indexes
  • business logic

11. Why Vision?

Vision is not SQL, not a document store, and not a classic graph database. It is a Natural Data Engine (NDE) designed to offer a simple, robust and predictable alternative to traditional systems.

Compared to SQL

  • no tables, no schema, no migrations
  • no complex joins
  • direct file storage
  • simple explicit relations
  • RAM‑first read performance

Compared to Document Stores

  • each cell is autonomous (UniCell)
  • native relations (no arbitrary embedding)
  • natural date‑based sharding
  • deterministic, self‑validated format

Compared to Graph Databases

  • simple relation levels (0, 1, 2…)
  • multi‑shard navigation
  • cycle‑resistant
  • no global graph rewrites

What Vision Provides

  • a very simple engine to understand
  • no dependencies, no server
  • atomic writes, crash‑safe
  • no corruption possible
  • human‑readable structure
  • ideal for forums, CMS, notes, graphs, knowledge bases

12. Conclusion

Vision is a natural, simple, stable and predictable data engine. UniCell provides universality. Wave provides coherence. Modules provide dynamics.

Vision is not a traditional database. Vision is a living architecture designed for developers who want full control with maximum robustness.
